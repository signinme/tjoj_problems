基于交易反欺诈的账户风险识别模型

问题背景

近年来，银行线上业务随着移动互联网的兴起而快速发展，互联网金融业务涉及多种欺诈风险，国家也密切关注账户资金流动的合规性风险，监管趋严，监管文件频出。如银发【2016】261号文、银发【2019】85号文、银保监办发【2021】49号文等。同时，2020年以来涉赌涉诈案件频发，公安部加大打击力度，要求银行对其账户做内部排查，及时关停可疑账户。为保障银行线上金融业务的平稳运行、风险可控、账户安全规范，提出了基于线上交易反欺诈的风险识别的需求。

本实验提供风险账户的基础信息、操作行为信息和交易行为信息，通过数据分析，从账户开户特征、设备操作特征、交易频次、交易时间、交易金额、地域分布等维度进行特征挖掘，建立有监督的机器学习模型，有效甄别存在高风险交易的账户。

数据说明

数据集分为train_with_label(训练数据集)、test_without_label(测试数据集)。

训练数据集中的文件包含：
正负样本标签(0/1)：train_label.csv
脱敏后的用户基础信息：train_base.csv
脱敏后的用户操作信息：train_op.csv
脱敏后的用户交易信息：train_trans.csv
字段说明文件：数据字典.doc

测试数据集的文件包含：
脱敏后的用户基础信息：test_base.csv
脱敏后的用户操作信息：test_op.csv
脱敏后的用户交易信息：test_trans.csv
提交结果文件样例：submit_example.csv

评分标准

采用auc对结果进行评分，评测机代码示例： 

from sklearn.metrics import roc_auc_score

y_true = [1, 0, 1, 1]

y_pred = [1,0, 1, 0]

score = roc_auc_score(y_true, y_pred) * 100

注：每人只有3次提交结果进行评测的机会，评测结果超过70分以上即可
